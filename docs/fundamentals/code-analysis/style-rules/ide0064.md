## <a name="overview"></a><span data-ttu-id="6e77f-116">Обзор</span><span class="sxs-lookup"><span data-stu-id="6e77f-116">Overview</span></span>

<span data-ttu-id="6e77f-117">Это правило обнаруживает структуры, содержащие одно или несколько полей `readonly`, а также содержит назначение для `this` за пределами конструктора.</span><span class="sxs-lookup"><span data-stu-id="6e77f-117">This rule detects structs that contain one ore more `readonly` fields and also contains an assignment to `this` outside of the constructor.</span></span> <span data-ttu-id="6e77f-118">Правило рекомендует преобразовать поля `readonly` в режим "не только для чтения", т. е. с доступом для записи.</span><span class="sxs-lookup"><span data-stu-id="6e77f-118">The rule recommends converting `readonly` fields to non-readonly, i.e. writable.</span></span> <span data-ttu-id="6e77f-119">Пометка таких полей структуры как readonly может привести к непредвиденному поведению, так как значение, присваиваемое полю, может измениться, когда `this` назначается за пределами конструктора.</span><span class="sxs-lookup"><span data-stu-id="6e77f-119">Marking such struct fields as readonly can lead to unexpected behavior, as the value assigned to the field can change when `this` is assigned outside the constructor.</span></span>

## <a name="example"></a><span data-ttu-id="6e77f-120">Пример</span><span class="sxs-lookup"><span data-stu-id="6e77f-120">Example</span></span>

```csharp
// Code with violations
struct MyStruct
{
    public readonly int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}

// Fixed code
struct MyStruct
{
    public int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}
```
