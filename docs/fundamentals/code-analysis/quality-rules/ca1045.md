## <a name="cause"></a><span data-ttu-id="a96ea-111">Причина</span><span class="sxs-lookup"><span data-stu-id="a96ea-111">Cause</span></span>

<span data-ttu-id="a96ea-112">Открытый или защищенный метод в открытом типе имеет параметр `ref`, принимающий примитивный тип, ссылочный тип или тип значения, не являющийся одним из встроенных типов.</span><span class="sxs-lookup"><span data-stu-id="a96ea-112">A public or protected method in a public type has a `ref` parameter that takes a primitive type, a reference type, or a value type that is not one of the built-in types.</span></span>

## <a name="rule-description"></a><span data-ttu-id="a96ea-113">Описание правила</span><span class="sxs-lookup"><span data-stu-id="a96ea-113">Rule description</span></span>

<span data-ttu-id="a96ea-114">Для реализации передачи типов по ссылке (с помощью ключевого слова `out` или `ref`) от разработчика требуется опыт работы с указателями, понимание отличия между типами значения и ссылочными типами и умение работать с методами с несколькими возвращаемыми значениями.</span><span class="sxs-lookup"><span data-stu-id="a96ea-114">Passing types by reference (using `out` or `ref`) requires experience with pointers, understanding how value types and reference types differ, and handling methods that have multiple return values.</span></span> <span data-ttu-id="a96ea-115">Кроме того, далеко не все понимают разницу между параметрами `out` и `ref`.</span><span class="sxs-lookup"><span data-stu-id="a96ea-115">Also, the difference between `out` and `ref` parameters is not widely understood.</span></span>

<span data-ttu-id="a96ea-116">Когда ссылочный тип передается "по ссылке", метод намеревается использовать этот параметр для возврата другого экземпляра объекта.</span><span class="sxs-lookup"><span data-stu-id="a96ea-116">When a reference type is passed "by reference," the method intends to use the parameter to return a different instance of the object.</span></span> <span data-ttu-id="a96ea-117">(Передача ссылочного типа по ссылке также называется использованием двойного указателя, указателем на указатель или двойным косвенным обращением.) При использовании соглашения о вызовах по умолчанию, которое передает "по значению", параметр, который принимает ссылочный тип, уже получает указатель на объект.</span><span class="sxs-lookup"><span data-stu-id="a96ea-117">(Passing a reference type by reference is also known as using a double pointer, pointer to a pointer, or double indirection.) Using the default calling convention, which is pass "by value," a parameter that takes a reference type already receives a pointer to the object.</span></span> <span data-ttu-id="a96ea-118">Указатель, а не объект, на который он указывает, передается по значению.</span><span class="sxs-lookup"><span data-stu-id="a96ea-118">The pointer, not the object to which it points, is passed by value.</span></span> <span data-ttu-id="a96ea-119">Передача по значению означает, что метод не может изменить указатель, чтобы он указывал на новый экземпляр ссылочного типа, но может изменить содержимое объекта, на который он указывает.</span><span class="sxs-lookup"><span data-stu-id="a96ea-119">Passing by value means that the method cannot change the pointer to have it point to a new instance of the reference type, but can change the contents of the object to which it points.</span></span> <span data-ttu-id="a96ea-120">Для большинства приложений это достаточно и дает желаемое поведение.</span><span class="sxs-lookup"><span data-stu-id="a96ea-120">For most applications this is sufficient and yields the behavior that you want.</span></span>

<span data-ttu-id="a96ea-121">Если метод должен возвращать другой экземпляр, используйте для этого возвращаемое значение метода.</span><span class="sxs-lookup"><span data-stu-id="a96ea-121">If a method must return a different instance, use the return value of the method to accomplish this.</span></span> <span data-ttu-id="a96ea-122">См. класс <xref:System.String?displayProperty=fullName> для различных методов, которые работают со строками и возвращают новый экземпляр строки.</span><span class="sxs-lookup"><span data-stu-id="a96ea-122">See the <xref:System.String?displayProperty=fullName> class for a variety of methods that operate on strings and return a new instance of a string.</span></span> <span data-ttu-id="a96ea-123">Использование этой модели позволяет вызывающему объекту решить, сохраняется ли исходный объект.</span><span class="sxs-lookup"><span data-stu-id="a96ea-123">By using this model, it is left to the caller to decide whether the original object is preserved.</span></span>

<span data-ttu-id="a96ea-124">Хотя возвращаемые значения являются наиболее распространенными и часто используются, правильное применение параметров `out` и `ref` требует среднего уровня навыков проектирования и программирования.</span><span class="sxs-lookup"><span data-stu-id="a96ea-124">Although return values are commonplace and heavily used, the correct application of `out` and `ref` parameters requires intermediate design and coding skills.</span></span> <span data-ttu-id="a96ea-125">Архитекторам, разрабатывающим библиотеки для широкого использования, не следует рассчитывать, что пользователи хорошо разбираются в использовании параметров `out` и `ref`.</span><span class="sxs-lookup"><span data-stu-id="a96ea-125">Library architects who design for a general audience should not expect users to master working with `out` or `ref` parameters.</span></span>

> [!NOTE]
> <span data-ttu-id="a96ea-126">При работе с параметрами, которые являются большими структурами, дополнительные ресурсы, необходимые для копирования этих структур, могут повлиять на производительность при передаче по значению.</span><span class="sxs-lookup"><span data-stu-id="a96ea-126">When you work with parameters that are large structures, the additional resources that are required to copy these structures could cause a performance effect when you pass by value.</span></span> <span data-ttu-id="a96ea-127">В таких случаях можно рассмотреть использование параметров `ref` или `out`.</span><span class="sxs-lookup"><span data-stu-id="a96ea-127">In these cases, you might consider using `ref` or `out` parameters.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="a96ea-128">Устранение нарушений</span><span class="sxs-lookup"><span data-stu-id="a96ea-128">How to fix violations</span></span>

<span data-ttu-id="a96ea-129">Чтобы устранить нарушение этого правила, вызванное типом значения, метод должен вернуть объект в качестве возвращаемого значения.</span><span class="sxs-lookup"><span data-stu-id="a96ea-129">To fix a violation of this rule that is caused by a value type, have the method return the object as its return value.</span></span> <span data-ttu-id="a96ea-130">Если метод должен возвращать несколько значений, перепроектируйте его для возврата одного экземпляра объекта, содержащего значения.</span><span class="sxs-lookup"><span data-stu-id="a96ea-130">If the method must return multiple values, redesign it to return a single instance of an object that holds the values.</span></span>

<span data-ttu-id="a96ea-131">Чтобы устранить нарушение этого правила, вызванное ссылочным типом, убедитесь, что желаемое поведение состоит в возвращении нового экземпляра ссылки.</span><span class="sxs-lookup"><span data-stu-id="a96ea-131">To fix a violation of this rule that is caused by a reference type, make sure that the behavior that you want is to return a new instance of the reference.</span></span> <span data-ttu-id="a96ea-132">Если это так, метод должен использовать его возвращаемое значение для этого.</span><span class="sxs-lookup"><span data-stu-id="a96ea-132">If it is, the method should use its return value to do this.</span></span>
