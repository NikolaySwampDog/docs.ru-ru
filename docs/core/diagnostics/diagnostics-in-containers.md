```dockerfile

RUN dotnet tool install --tool-path /tools dotnet-counters
RUN dotnet tool install --tool-path /tools dotnet-dump

...

# In final stage
# Copy diagnostics tools
WORKDIR /tools
COPY --from=build /tools .
```

<span data-ttu-id="45e8f-116">В качестве альтернативы пакет SDK для .NET Core можно установить в контейнере, если это необходимо для установки средств CLI.</span><span class="sxs-lookup"><span data-stu-id="45e8f-116">Alternatively, the .NET Core SDK can be installed in a container when needed in order to install the CLI tools.</span></span> <span data-ttu-id="45e8f-117">Учитывайте, что при установке пакета SDK для .NET Core среда выполнения .NET Core будет переустановлена.</span><span class="sxs-lookup"><span data-stu-id="45e8f-117">Be aware that installing the .NET Core SDK will have the side-effect of reinstalling the .NET Core runtime.</span></span> <span data-ttu-id="45e8f-118">Поэтому обязательно установите версию пакета SDK, соответствующую среде выполнения в контейнере.</span><span class="sxs-lookup"><span data-stu-id="45e8f-118">So be sure to install the version of the SDK that matches the runtime present in the container.</span></span>
