## <a name="nullable"></a><span data-ttu-id="36aee-199">Допускает значения NULL</span><span class="sxs-lookup"><span data-stu-id="36aee-199">Nullable</span></span>

<span data-ttu-id="36aee-200">Параметр **Nullable** позволяет указать контекст, допускающий значение NULL.</span><span class="sxs-lookup"><span data-stu-id="36aee-200">The **Nullable** option lets you specify the nullable context.</span></span>

```xml
<Nullable>enable</Nullable>
```

<span data-ttu-id="36aee-201">Аргумент должен иметь одно из следующих значений: `enable`, `disable`, `warnings` или `annotations`.</span><span class="sxs-lookup"><span data-stu-id="36aee-201">The argument must be one of `enable`, `disable`, `warnings`, or `annotations`.</span></span> <span data-ttu-id="36aee-202">Аргумент `enable` разрешает контекст, допускающий значение NULL.</span><span class="sxs-lookup"><span data-stu-id="36aee-202">The `enable` argument enables the nullable context.</span></span> <span data-ttu-id="36aee-203">При указании `disable` контекст, допускающий значение NULL, будет отключен.</span><span class="sxs-lookup"><span data-stu-id="36aee-203">Specifying `disable` will disable the nullable context.</span></span> <span data-ttu-id="36aee-204">Аргумент `warnings` включает контекст предупреждения, допускающий значение NULL.</span><span class="sxs-lookup"><span data-stu-id="36aee-204">When providing the `warnings` argument the nullable warning context is enabled.</span></span> <span data-ttu-id="36aee-205">При указании аргумента `annotations` будет включен контекст заметок, допускающий значение NULL.</span><span class="sxs-lookup"><span data-stu-id="36aee-205">When specifying the `annotations` argument, the nullable annotation context is enabled.</span></span>
